Getting Started with Git: A Beginner's Guide

Introduction to Git

Git is a distributed version control system that helps you track changes in your files and collaborate with others on projects. It allows multiple users to work on the same files simultaneously while keeping track of every change made. In this blog, we'll dive into the basics of Git, focusing on its installation, configuration, basic commands, and essential concepts.

Version Control Systems

A version control system (VCS) tracks changes to files over time. It enables you to revert to previous versions, compare changes, and collaborate efficiently. Git is one of the most popular VCS, but there are others like Mercurial, SVN, and Perforce. However, we'll primarily focus on Git and its integration with GitHub.

Git Installation

Before diving into Git, you need to ensure it's installed on your system. To check, open your terminal (Command Prompt, Git Bash, or Terminal) and type:
git --version

If Git is not installed, follow these steps to download and install it:

  1. Visit the official Git website.
  2. Download the appropriate version for your operating system.
  3. Run the installer and follow the instructions.

Checking whether git is installed in your machine.

Configuring Git

Once Git is installed, you need to configure your username and email address to identify your commits. Use the following commands:
git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"

Configuring your username and email address.

Git Repository

A Git repository is a storage space where your project's files and revision history are stored. To initialize a Git repository in your project directory, use:
git init

Initializing a Git repository

Link Your Local Repository with GitHub:

Copy the URL of your GitHub repository. It should look like
"https://github.com/username/repository.git"
In your terminal, set the remote URL for your Git repository using the following command (replace with the URL you copied):
git remote add origin "URL"

Creating a repository on GitHub and linking it with Git

Staging and Committing Changes

In Git, changes are staged before they are committed to the repository. Untracked files are those not yet staged or committed. To stage files, use:
git add "filename" where "filename" is the name of your file or use:
git add . to add all the files in your directory

Adding all the files in your directory

To commit changes, use:
git commit -m "Your commit message"

Committing changes

Push Your Local Repository to GitHub:

Finally, Push your commits to the remote repository on GitHub:
git push -u origin master

Git Branching

Branches in Git are independent lines of development that allow you to work on features or fixes without affecting the main codebase. They're crucial for collaboration and maintaining a clean codebase.

Creating and Managing Branches

To create a new branch, use:
git checkout -b new_branch_name

Creating a new branch

To view all branches:
git branch

Checking branches

To switch between branches:
git checkout branch_name or git switch branch_name

Switching branches

Merging Branches

To merge a branch into the active branch, use:
git merge branch_name

Deleting Branches

To delete a branch, use:
git branch -d branch_name

Deleting branches

Next Steps

Now that you've grasped the basics of Git, consider learning more about collaborative workflows, branching strategies, or exploring platforms like GitHub for hosting your repositories and collaborating with others. Dive deeper into Git by exploring topics like rebasing, resolving merge conflicts, and Git hooks. Happy coding!
